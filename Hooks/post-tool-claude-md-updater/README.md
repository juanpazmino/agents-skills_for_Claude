# post-tool-claude-md-updater

A drop-in folder that adds a `PostToolUse` hook to Claude Code. After every `Write` or `Edit` tool call, it uses the Claude API to generate a one-line summary of what changed and prepends it to your project's `CLAUDE.md` under a `## Recent Changes` section.

## Folder Structure

```
your_project/
├── .claude/
│   └── settings.example.json   # Hook registration (contains $PWD placeholder)
│   └── settings.local.json     # Generated by init script (actual absolute path)
├── hooks/
│   └── update_claude_md.js     # Hook logic — summarizes changes and updates CLAUDE.md
├── scripts/
│   └── init-hook.js            # Setup script — resolves $PWD to an absolute path
└── package.json                # Dependencies + init script
```

## Usage

There are three ways to use this hook depending on your scope.

---

### Option A — Local (personal, this project only)

The hook applies only to this project and is not committed to version control.

#### 1. Copy the folder into your project root

```bash
cp -r post-tool-claude-md-updater/hooks        your_project/hooks
cp -r post-tool-claude-md-updater/.claude      your_project/.claude
cp -r post-tool-claude-md-updater/scripts      your_project/scripts
cp    post-tool-claude-md-updater/package.json your_project/package.json
```

#### 2. Install dependencies

```bash
npm install
```

#### 3. Run the setup script

From your project root:

```bash
npm run init
```

This replaces the `$PWD` placeholder in `.claude/settings.example.json` with the real absolute path of your project and writes it to `.claude/settings.local.json` (gitignored, never committed).

```
✅ Successfully created .claude/settings.local.json
   Replaced $PWD with: /path/to/your/project
```

---

### Option B — Project (shared with your team)

The hook and its registration are committed so every team member gets it automatically. Use a relative path in `.claude/settings.json` so it resolves correctly on any machine.

#### 1. Copy the folder into your project root

```bash
cp -r post-tool-claude-md-updater/hooks        your_project/hooks
cp -r post-tool-claude-md-updater/.claude      your_project/.claude
cp -r post-tool-claude-md-updater/scripts      your_project/scripts
cp    post-tool-claude-md-updater/package.json your_project/package.json
```

#### 2. Install dependencies

```bash
npm install
```

#### 3. Register the hook using a relative path

Create or update `.claude/settings.json` in your project root:

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "node hooks/update_claude_md.js"
          }
        ]
      }
    ]
  }
}
```

#### 4. Commit the files

```bash
git add .claude/settings.json hooks/update_claude_md.js package.json
git commit -m "Add post-tool CLAUDE.md updater hook"
```

> Make sure `.claude/settings.local.json` is in your `.gitignore` so personal overrides are never committed.

---

### Option C — Global (personal, all projects)

One shared hook file in `~/.claude/hooks/` applies to every Claude Code session on this machine.

#### 1. Copy the hook file and install dependencies

```bash
mkdir -p ~/.claude/hooks
cp post-tool-claude-md-updater/hooks/update_claude_md.js ~/.claude/hooks/
cd ~/.claude/hooks && npm install @anthropic-ai/sdk
```

#### 2. Register the hook in your global settings

Add the following to `~/.claude/settings.json` (create it if it doesn't exist):

```json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [
          {
            "type": "command",
            "command": "node /Users/YOUR_USERNAME/.claude/hooks/update_claude_md.js"
          }
        ]
      }
    ]
  }
}
```

Replace `/Users/YOUR_USERNAME` with your actual home directory path.

## How it works

After every `Write` or `Edit` tool call, `update_claude_md.js` receives the tool payload via stdin, calls the Claude API (Haiku model) to generate a concise summary of the change, then inserts it at the top of the `## Recent Changes` section in your project's `CLAUDE.md`.

The result looks like this in your `CLAUDE.md`:

```markdown
## Recent Changes

- **2026-03-06** `src/auth.js` — Added JWT authentication middleware
- **2026-03-06** `src/db.js` — Replaced raw SQL queries with Knex query builder
```

**Safeguards built in:**
- Skips edits to `CLAUDE.md` itself to prevent infinite loops
- Skips `.git/` internals
- Exits silently (code 0) on any error — never blocks Claude's work
- Exits silently if `ANTHROPIC_API_KEY` is not set
- Never sends sensitive file content to the API — blocks `.env`, `.envrc`, private keys, certs, SSH keys, and database files

## Requirements

- Node.js 18+
- `ANTHROPIC_API_KEY` available in your environment (Claude Code inherits it automatically)
